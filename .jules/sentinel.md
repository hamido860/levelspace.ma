## 2026-04-20 - [CRITICAL] Fix authorization bypass in Supabase proxy\n**Vulnerability:** The public `/api/supabase/proxy` endpoint initialized the server-side Supabase client using `SUPABASE_SERVICE_ROLE_KEY` if available. This is a critical security vulnerability, as it allows any unauthenticated user to make POST requests to the proxy and bypass Row Level Security (RLS) entirely, giving them full admin access to the database.\n**Learning:** Backend proxies meant to handle arbitrary client requests must initialize their DB client with an ANON_KEY or explicitly forward the client's JWT to preserve Row Level Security. Never map a public proxy endpoint to a Service Role DB client.\n**Prevention:** Initialize Supabase in `server.ts` with `SUPABASE_ANON_KEY` to ensure the proxy correctly enforces RLS.
