## $(date +%Y-%m-%d) - Secured Serverless Proxy against Unauthenticated Abuse
**Vulnerability:** The Vercel serverless proxy endpoint `/api/nvidia-proxy` was completely unauthenticated, allowing any external user to bypass application frontend controls and directly hit the server-side proxy. This allowed malicious actors to consume the application's sensitive `NVIDIA_API_KEY` quota without being logged in, essentially forming an SSRF and resource-exhaustion vulnerability.
**Learning:** Proxy endpoints designed to hide API keys from the browser must also enforce the same (or stricter) authentication mechanisms as the rest of the application. Leaving them open negates the value of hiding the key. We learned that `requireAuthenticatedUser` from `aiCommandCenter.ts` can be effectively applied to generic API endpoints to enforce user session checks via Supabase auth tokens. We also learned how to allow custom key overrides (`X-Nvidia-Api-Key`) securely so users can supply their own keys without exposing the system's key.
**Prevention:** In the future, any new backend endpoint created to proxy requests to a third-party API must be built with `requireAuthenticatedUser` by default. Never assume a route is safe just because its client-side caller is hidden behind a UI element. Always protect the route explicitly.
