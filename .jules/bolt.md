## 2024-05-24 - Memoizing Dexie live queries to prevent cascading re-renders
**Learning:** When using Dexie's `useLiveQuery` hook, deriving new structures (like arrays using `.filter()`, `.map()`, or objects using `Object.fromEntries()`) creates new object references on every evaluation. Because `useLiveQuery` is sensitive to any database update and causes re-evaluations, failing to memoize these derived derivations breaks React's referential equality, leading to severe performance bottlenecks through cascading re-renders in child components.
**Action:** Always wrap derived structures generated from `useLiveQuery` (or from the `toArray()` / `.map()` of Dexie tables) in a `useMemo` hook, ensuring that the dependency array includes the source query result (e.g., `dbSettings` or `dbModules`). This preserves object references and prevents unnecessary re-renders.
