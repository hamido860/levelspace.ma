## 2025-05-18 - Fix unauthenticated SSRF in NVIDIA proxy
**Vulnerability:** The Vercel serverless function `handleNvidiaProxy` exposed an endpoint that consumed the backend `NVIDIA_API_KEY` but lacked any authentication checks. This allowed unauthenticated, cross-origin abuse of the endpoint.
**Learning:** Serverless proxy routes that forward to billing-sensitive third-party APIs must always have identical authentication checks as internal RPC calls.
**Prevention:** Always wrap Vercel node function handlers in an `await requireAuthenticatedUser(req)` block before proceeding to process the request payload or secrets.
## 2025-02-27 - [CRITICAL] Fix unauthenticated platform AI credit resource exhaustion
**Vulnerability:** The AI endpoints (`handleAIGenerate`, `handleAIExplain`, `handleAILessonBlocks`) in `src/server/api/aiHandlers.ts` did not strictly enforce authentication for the `platform` credential mode, allowing unauthenticated requests to use platform AI credits without a valid user ID.
**Learning:** Context resolvers like `resolveCredentialContext` successfully returned the context (`credentialMode: "platform"`) but deliberately do not throw HTTP errors, as they are pure state evaluators. The route handlers were incorrectly failing to check the resolved context for valid authentication (missing `if (credentials.credentialMode === "platform" && !credentials.userId)` check), leading to SSRF and missing authentication vulnerabilities.
**Prevention:** Always follow up calls to context/state resolvers (like `resolveCredentialContext`) with explicit conditional checks in the route handler logic to throw the necessary 401 Unauthorized HTTP errors before processing the request.
