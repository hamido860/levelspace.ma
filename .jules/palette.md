## $(date +%Y-%m-%d) - Add ARIA Labels and Keyboard Access to Interactive Topbar Elements
**Learning:** Found a recurring pattern in the Topbar where icon-only buttons (`RefreshCw`, `Moon`/`Sun`, `Globe`, `Settings`, `Bell`) lacked accessible labels (`aria-label`) and explicit keyboard focus indicators. Furthermore, interactive `div` elements acting as buttons (like the profile section) did not have the correct `role="button"`, `tabIndex={0}`, and `onKeyDown` handlers to support keyboard activation (Enter/Space), preventing keyboard users from accessing profile or settings functionalities.
**Action:** Always add descriptive `aria-label`s to icon-only buttons. When using `div` elements for interactions, ensure they are given `role="button"`, `tabIndex={0}`, and handle `e.preventDefault()` along with execution for 'Enter' or ' ' keys. To provide visual focus feedback, standardly apply `focus:outline-none focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-1` utility classes.
