## 2025-05-18 - Fix unauthenticated SSRF in NVIDIA proxy
**Vulnerability:** The Vercel serverless function `handleNvidiaProxy` exposed an endpoint that consumed the backend `NVIDIA_API_KEY` but lacked any authentication checks. This allowed unauthenticated, cross-origin abuse of the endpoint.
**Learning:** Serverless proxy routes that forward to billing-sensitive third-party APIs must always have identical authentication checks as internal RPC calls.
**Prevention:** Always wrap Vercel node function handlers in an `await requireAuthenticatedUser(req)` block before proceeding to process the request payload or secrets.
## 2024-05-14 - Missing Authorization Check After Context Resolution
**Vulnerability:** Unauthenticated clients could use platform AI credits for `handleAIGenerate`, `handleAIExplain`, and `handleAILessonBlocks` endpoints, leading to Server-Side Request Forgery (SSRF) and potential resource exhaustion.
**Learning:** The `resolveCredentialContext` helper in `aiHandlers.ts` is designed to strictly evaluate and return credential state (e.g., `{ credentialMode: "platform", userId: undefined }`). It does not enforce authorization or throw errors. The downstream route handlers assumed that calling the context resolver implied authentication, failing to evaluate the resulting state.
**Prevention:** In backend architectures where context resolvers return state rather than asserting it, downstream route handlers must explicitly check the resolved credentials (e.g., `if (credentials.credentialMode === "platform" && !credentials.userId)`) and return HTTP 401 Unauthorized errors to enforce security boundaries.
